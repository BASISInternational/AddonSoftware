2. WHAT IS NEW IN THIS RELEASE(top of page)

Version 21.10 takes advantage of the technological advancements provided by Barista® and BBj® to deliver an innovative software package known for its structural quality. Together with Barista, AddonSoftware offers a leap forward in the efficiency of software customization and the creation of vertical applications. AddonSoftware continues its history of constant improvement - additional features, increased functionality, and updated technology - in this release.
  • Preview features for version 22.00
  • New for version 21.00

    • General
      • Addon installs and upgrades are now version controlled using Git. Features of this new Git-based model include:
        • Installs and upgrades automatically create the recommended version-neutral file structure.
        • Facilitates making Addon customizations directly in the Addon file structure, eliminating the need for a separate "mods project."
        • The Upgrade Wizard uses Barista's "Saved Selection Options" feature to save and quickly restore grid entries when resuming/restarting.
        • The new Packager/Extractor utility simplifies moving a Barista/Addon instance between production and test servers.
        • Automatically incorporates customizations from mods projects directly into Addon.
      • All address and city fields have been expanded to 30 characters.
      • Email address, subject, and message fields in the Report Control forms have been expanded.
    • Accounting
      • Added a new column to the AP Payment Selection Report to flag ACH vendors, and show Total ACH and Total Check payments in final report totals.
      • Added Barista's "Saved Selection Option" feature to the Payment Selection Entry form.
      • Report Control Recipients form can be accessed from Vendor Maintenance.
      • Bank Account field in the Bank Accounts and Vendors tables has been encrypted, and new VIEWs created to show account last 4 if accessing data externally (i.e., via SQL from outside of Addon).
      • Added aging criterion information to the Credit Maintenance form and the Totals tab on Order/Invoice Entry form.
      • Modified the AR Aging Report to require 30-day fixed periods, and always aging credits and future invoices when the "Update Customer Aging Summary" option is selected.
      • A new option is available in the Customers, Credit Maintenance, and Order/Invoice Entry forms that allows users to refresh the customer's aged balances on demand. The option is enabled or disabled via a new "On-demand Aging?" parameter.
      • The new Default Age By parameter supplies the default aging basis (I=Invoice Date, D=Due Date) for the Aging Report, Age Transactions, and Statements tasks as well as for back-end aging updates when no aging basis has yet been established for a given customer.
      • The Customer form's Aging/Sales Summary tab has a new drill down on the Last Payment Date to quickly see how the payment was applied.
      • Report Control Recipients form can be accessed directly from Customer Maintenance.
      • Description for Chart of Accounts breaks expanded to 50 characters.
    • Distribution
      • Interface with Avalara's AvaTax sales tax service:
        • Send Orders to AvaTax for estimated tax.
        • Send Invoices to AvaTax for final tax calculation.
        • Commit/finalize transactions during Sales Register/Update.
        • Automatically updates new Customer Sales Tax Exemptions table.
        • Associate items with AvaTax tax service codes via Item Master, Product Type, Item Type, and/or Item Class .
        • Committed transaction history and complete JSON response available for research, reporting, or auditing.
        • Supports using AvaTax sales tax service together with traditional Addon sales tax codes.
      • Added a "Sequence" ListButton to the Invoice Printing form so that invoices can be printed in Customer ID sequence or Invoice Number sequence.
      • New hand-held application for Physical Inventory Counting to use instead of or in conjunction with traditional count sheets.
      • Expanded Item Class to 6 characters.
      • Expanded Customer Pricing Code to 6 characters.
    • Testing
      • This version passes all QF-Test suites.

  • For a list of continued improvements in all modules, refer to the Readme/Relnotes and Fixes link on the BASIS product download page.
3. KNOWN ISSUES(top of page)
  • BBj version 21.10 and the Barista Application Framework version 21.10 are both required to run AddonSoftware version 21.10.
  • When running under Linux, the KDE desktop environment must be used.
4. SUPPORT(top of page)

If you purchased your AddonSoftware product from a distributor, reseller, or OEM, please contact them for technical support. They will answer your question or contact us on your behalf.

When submitting a technical support question, please provide the following information:
  1. The product serial number.
  2. The name of the product and the product's version number.
  3. A complete description of the problem, including the product error and the TCB(10) value, if available.
  4. A screen shot of the problem.
Other pertinent information includes:
  1. The operating system name, JRE, level and version.
  2. Is the error consistently reproducible, or is it sporadic?
  3. Is the error isolated to a specific machine or user?
  4. Is this a new installation or has the system been in place and operating normally?
  5. Is the problem network related/specific?
