2. WHAT IS NEW IN THIS RELEASE(top of page)

Version 22.01 takes advantage of the technological advancements provided by Barista® and BBj® to deliver an innovative software package known for its structural quality. Together with Barista, AddonSoftware offers a leap forward in the efficiency of software customization and the creation of vertical applications. AddonSoftware continues its history of constant improvement - additional features, increased functionality, and updated technology - in this release.
  • Preview for version 23.00
    • Accounting
      • New "Print Check" button in Manual Check Entry allows printing manual checks on-demand.
      • Scanned AP Invoice images can now be attached to any invoice, regardless of whether or not Payment Authorization is used, and new parameters can be set to warn on registers and/or disallow updates if an image has not been linked to an invoice.
    • Distribution
      • Picking Lists, Order Acknowledgements and Invoices now show Cash customer manual ship-to addresses in both the Bill-To and Ship-To address boxes.
      • Order Entry can be filtered to present only Quotes, or only Orders, or both.
      • A new lookup in Order and Invoice Entry shows historical manual ship-to addresses, and a new option allows creating a Customer ship-to address from the manual ship-to address.
      • The Customer Ship-To Addresses form has a new "Use Manual Ship-To" option that lets the user create a new ship-to address from a manual ship-to address previously used for the customer.
  • New for version 22.00
    • Accounting
      • Four new AR parameters allow users to specify the default period sizes for all AR agings. The AR Aging Report is further enhanced so users can override the defaults to specify unique period sizes for each report.
      • The Start Date and Period End Dates in the Fiscal Calendar are now full YYYYMMDD format to provide accuracy and flexibility, particularly when working in a non-calendar fiscal year.
      • AP Checks can apply a stored electronic signature regardless of whether or not Payment Authorization is used. Default signature is supplied and can be overridden.
      • Modified Bank Reconciliation to skip listing checks and other transactions later than the current statement date.
      • The Bank Account Codes table now keeps track of the next check number, and uses it as the default for AP Check Printing.
      • The Cash Requirements Report has a new option to include and flag, or exclude, vendors paid via Automated Clearing House (ACH).
      • Improved filtering in GL Transaction History Report so Audit Number filter can be coupled with Account Number and/or Journal ID filters.
      • Added the GL Account Description to the Export Format of the GL Transaction History Report.
      • Enhanced the Order lookup on the Customer form to include Salesperson and Credit Flag.
      • Added Audit Control Number drilldown to the "Detail" view query on the GL Account Summary Activity form.
      • Enhanced the AP Export 1099 To Text form so user can specify both a server and client-side save directory.
      • Different checking accounts can use the same check numbers.
      • Columns in the Bank Reconciliation Check Detail and Deposit/Other Transactions grids can now be sorted for faster processing.
      • Manual Check Entry now permits users to select multiple invoices for payment from the Open Invoice lookup grid.
      • Accounts Payable has been enhanced so that checks for selected vendors can be limited to paying a single invoice.
      • The General Ledger Daily Detail Update can now post to the suspense account if applicable. GL reports and month/year-end close processes show if suspense account balance is not zero, and year-end close is prohibited.
    • Distribution
      • New "Change Customer" Additional Option in Order Entry allows users to change the Customer ID on an open order/quote.
      • Send customers an Order Acknowledgement via email or fax. Supports generating acknowledgements automatically or manually, on demand or in batch mode. Interfaces with Addon Report Control.
      • Added Pricing Basis and Amount fields to the Pricing Table form to allow quantity price breaks based on a specific amount as well as a percentage.
      • Modified the Open Order Detail report to allow more intuitive and granular selections for Orders vs Quotes.
      • Added the Fiscal Period abbreviations to row labels on Sales Analysis screens for more clarity.
      • Added an Amount Received column to the Purchase Receipt query in Purchase Order Invoice Entry to make finding the correct PO/Receipt easier when there are multiple receivers.
      • Added Report Control Status (Email/Fax/None) to Order/Invoice Entry and Purchase Order Entry.
      • Added an "Include Zero Balance Items" option to the Inventory Stock Status Report.
      • Focus directly on Freight Amount in Order/Invoice Entry's Totals tab rather than Discount Amount, since the latter isn't often changed.
      • Added the Order or Invoice number to the saved document name for on-demand Pick Lists and Invoices, respectively.
      • In grids where warehouse is entered before the item, restrict the IV Item lookup to show only those items for the currently entered warehouse.
      • Added option to Inventory Physical Count Register to control whether the count should be set to zero vs the current freeze quantity, when a count is not entered for an item.
      • Refactored the Purchase Order Calendar to use the new BBjCalendarWidget. Allows users to set their own colors for Workday, Holiday, Closed.
      • Modified the Order Picking List to automatically reprint when changes are made, and clearly identify those changes on the "Reprinted" Picking List.
      • Invoice Bill-to and Ship-to addresses are saved for accuracy on Historical Invoices or Invoice History Inquiry.
      • Modified Order/Invoice Entry to use last warehouse entered for that order/invoice as default warehouse for next detail line entered for that order/invoice.
    • Manufacturing
      • Disallow deletion of Operations records in Bill of Materials and Work Order Entry when the Operation is referenced in a Subcontract or Materials record.
      • When releasing a Work Order, automatically select all checkboxes (Release Work Order, Print Traveler, Print Pick List).
      • The Shop Floor Calendar uses the new BBjCalendarWidget and has added functionality so user can display hours for more than one operations code at a time, add/edit hours for a range of days, and control the display color for each operation.
  • Testing
    • This version passes all QF-Test suites.

  • For a list of continued improvements in all modules, refer to the Readme/Relnotes and Fixes link on the BASIS product download page.

    4. SUPPORT(top of page)

    If you purchased your AddonSoftware product from a distributor, reseller, or OEM, please contact them for technical support. They will answer your question or contact us on your behalf.

    When submitting a technical support question, please provide the following information:
    1. The product serial number.
    2. The name of the product and the product's version number.
    3. A complete description of the problem, including the product error and the TCB(10) value, if available.
    4. A screen shot of the problem.
    Other pertinent information includes:
    1. The operating system name, JRE, level and version.
    2. Is the error consistently reproducible, or is it sporadic?
    3. Is the error isolated to a specific machine or user?
    4. Is this a new installation or has the system been in place and operating normally?
    5. Is the problem network related/specific?
