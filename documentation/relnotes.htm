2. WHAT IS NEW IN THIS RELEASE(top of page)

Version 23.00 takes advantage of the technological advancements provided by Barista® and BBj® to deliver an innovative software package known for its structural quality. Together with Barista, AddonSoftware offers a leap forward in the efficiency of software customization and the creation of vertical applications. AddonSoftware continues its history of constant improvement - additional features, increased functionality, and updated technology - in this release.
  • New for version 23.00
    • Accounting
      • New "Print Check" button in Manual Check Entry allows printing manual checks on-demand.
      • Scanned AP Invoice images can now be attached to any invoice, regardless of whether or not Payment Authorization is used, and new parameters can be set to warn on registers and/or disallow updates if an image has not been linked to an invoice.
      • Loading/viewing scanned AP Invoice images now display inside a BBjHtmlView control rather than an external browser.
      • Added a Vendor Pay-To Address additional option to the Vendor master form for when checks to the vendor are sent to an address different than the vendor's main address.
      • The Vendor Master has a new Payment History Inquiry option for viewing all checks and related invoices for the vendor.
      • The Bank Account Code table contains two new fields: one to control whether or not to include a MICR band at the bottom of AP Checks and On-demand Manual Checks, and the other to specify how to format the bank account number. The bank account number format has a drilldown to view valid MICR characters.
      • New filters and drill downs in the Vendor form's Historical Invoices query provide a link to the new Payment History Inquiry, so users can drill from invoices to payments/adjustments, then to check detail.
      • A new Payment History option on the Customer form shows payments, with a drill down from any given payment to the payment details/invoices paid. Drill on a given invoice to launch the AR Invoice History Inquiry form. The Payment History has also been added to the Customers node in Expresso Search.
      • A new right-click context menu option on the Historical Invoices query shows all the GL transactions associated with the selected Invoice.
      • A new AR Invoices query shows all, open, or paid invoices for a customer, and a right-click context menu option shows all the GL transactions associated with the selected Invoice.
      • The AR Simple Invoice update writes the entry header/detail records to a new set of archival files before removing them from the entry files. A new Archived AR Invoices form allows users to view and print the archival invoices.
      • The AP Invoice Number has been expanded from 10 to 20 characters.
    • Distribution
      • Picking Lists, Order Acknowledgements and Invoices now show Cash customer manual ship-to addresses in both the Bill-To and Ship-To address boxes.
      • Order Entry can be filtered to present only Quotes, or only Orders, or both.
      • A new lookup in Order and Invoice Entry shows historical manual ship-to addresses, and a new option allows creating a Customer ship-to address from the manual ship-to address.
      • The Customer Ship-To Addresses form has a new "Use Manual Ship-To" option that lets the user create a new ship-to address from a manual ship-to address previously used for the customer.
      • Barcodes for the customer, order, and detail line items can now be included on the Sales Order Picking List.
      • When printing for all warehouses, Picking List items are sorted by warehouse.
      • The new Order Fulfillment task lets users enter picking, packing and shipping information for an order, and print a legitimate Packing List rather than using the Pick List. After an order fulfillment is completed, the information can either be pulled into Invoice Entry, or pushed into Invoice Entry using the Fulfillment Report.
      • Increased the maximum allowed size of an AP Invoice number from 10 to 20 characters.
    • Manufacturing
      • Work Order comments can now be included on the Shop Floor Picking List.
  • Testing
    • This version passes all QF-Test suites.

  • For a list of continued improvements in all modules, refer to the Readme/Relnotes and Fixes link on the BASIS product download page.

    4. SUPPORT(top of page)

    If you purchased your AddonSoftware product from a distributor, reseller, or OEM, please contact them for technical support. They will answer your question or contact us on your behalf.

    When submitting a technical support question, please provide the following information:
    1. The product serial number.
    2. The name of the product and the product's version number.
    3. A complete description of the problem, including the product error and the TCB(10) value, if available.
    4. A screen shot of the problem.
    Other pertinent information includes:
    1. The operating system name, JRE, level and version.
    2. Is the error consistently reproducible, or is it sporadic?
    3. Is the error isolated to a specific machine or user?
    4. Is this a new installation or has the system been in place and operating normally?
    5. Is the problem network related/specific?
