1. OVERVIEW(top of page)

AddonSoftware® is a cost-effective, modular, full-featured and fully integrated enterprise resource planning solution for such mid-market industries as Accounting, Distributing, and Manufacturing.
  • AddonSoftware Accounting modules
    • Accounts Payable
    • Accounts Receivable
    • General Ledger

  • AddonSoftware Distribution modules
    • Inventory
    • Purchase Orders
    • Sales Orders
    • Sales Analysis

  • AddonSoftware Manufacturing modules
    • Bill of Materials
    • Shop Floor Control

2. WHAT IS NEW IN THIS RELEASE(top of page)

Version 25.10 takes advantage of the technological advancements provided by Barista® and BBj® to deliver an innovative software package known for its structural quality. Together with Barista, AddonSoftware offers a leap forward in the efficiency of software customization and the creation of vertical applications. AddonSoftware continues its history of constant improvement - additional features, increased functionality, and updated technology - in this release.
  • Preview for version 26.00
    • General
      • Added "Active Codes Only" and "Inactive Codes Only" filters to all code lookup.
    • Accounting
      • Provided a lookup for AR Label Codes.
    • Distribution
      • Created an optional Dropship virtual warehouse for tracking dropship orders for stock items.
      • Added a Return Of Goods feature that provides an "all in one" place to reverse PO receipts.
      • Added option to PO Receipt Register to either print, or NOT print, zero receipt items.
      • Allow defaulting the Order Fulfillment quantity picked to the shipped quantity.
      • Created new ARM_CUSTITEMS Customer Item Number table to establish the relationship between Addon's Item IDs and a customer's Item Numbers.
    • Manufacturing
  • New for version 25.00
    • General
      • File verify the existing STATE_CODE Element Type with the new ADC_STATECODE Postal State Codes table.
      • A new "Job Numbers" Expresso search was added under the "Customer" search.
    • Accounting
      • Users can now designate the name and save location for both ACH (NACHA) and Positive Pay files. Also created a Positive Pay example program for Washington Trust Bank.
      • The Daily Detail Register was modified to NOT require a password override to run the updated when a posting is to a locked period and the Journal ID allows posting to locked periods, nor when a posting is to a closed fiscal year and the Journal ID allows posting to closed fiscal years.
    • Distribution
      • New Order Entry Backordered Items query shows orders containing backordered items, along with available quantities at specified vs. all warehouses. Drill into a given order to see other items on the order. This information is also available in report form from the Inventory Reports menu, Backordered Items Report.
      • Provide a warning during Purchase Requisition Entry if a referenced Work Order is being closed complete.
      • The kitting feature added in version 24.00 has been enhanced so kits can be either priced or non-priced.
      • A "Total Whse Availability" Addition option was added to Order/Invoice Entry to launch the Item Price/Availability query for the selected item.
      • Order Fulfillment was modified to allow declaring an Order "All Packed" even when an Item's picked quantity is less than the order quantity.
      • "NS Item ID" and "Item ID" columns have been added to the Purchase Order Invoice Entry detail grid using the values from POT_RECDET.
      • Purchase Requisition Entry and Purchase Order Entry have been modified to populate the detail grid for dropships that are coming from a Sales Order.
    • Manufacturing

  • For a list of continued improvements in all modules, refer to the Readme/Relnotes and Fixes link on the BASIS product download page.

    4. SUPPORT(top of page)

    If you purchased your AddonSoftware product from a distributor, reseller, or OEM, please contact them for technical support. They will answer your question or contact us on your behalf.

    When submitting a technical support question, please provide the following information:
    1. The product serial number.
    2. The name of the product and the product's version number.
    3. A complete description of the problem, including the product error and the TCB(10) value, if available.
    4. A screen shot of the problem.
    Other pertinent information includes:
    1. The operating system name, JRE, level and version.
    2. Is the error consistently reproducible, or is it sporadic?
    3. Is the error isolated to a specific machine or user?
    4. Is this a new installation or has the system been in place and operating normally?
    5. Is the problem network related/specific?
